features

一个工具，搞定所有整理

GAME_DETECT

智能游戏识别

自动扫描 Steam、Epic、GOG、暴雪、育碧等主流启动器路径，结合体积 + exe + 目录结构启发式识别，无漏报。

SAVE_SYNC

存档跟随迁移

在 Documents、AppData、Saved Games 里模糊匹配存档目录，与游戏本体一起搬走，存档一条不丢。

JUNCTION

目录联接无感知

文件搬走后在原位建 NTFS Junction，启动器和游戏内部路径全部不变，零配置即用。

AI_CLASSIFY

资料分类归集

按文件后缀（.safetensors .gguf .pt …）和目录名关键字，自动区分模型权重、训练数据集、知识库文档四个类别。

DRY_RUN

先预览再执行

默认 dry-run 模式生成 plan.json 整理方案，审核无误后加 -Apply 参数再实际移动。

ROLLBACK

一键回滚

每次操作都写 op-log.json，随时反向执行：删 Junction → robocopy 复制回原位 → 清理新位置。

how_to_use

三步开始整理

01

下载 & 编辑配置

把 PCOrganizer 文件夹复制到目标电脑，用文本编辑器打开 config.psd1，填写游戏迁移目标盘和资料根目录。也可以用下方配置生成器直接下载。

# config.psd1 关键项
GameTargets    = @('D:\Games', 'E:\Games')
GameMinSizeGB  = 1
02

以管理员身份运行

右键 PowerShell → 以管理员身份运行，进入脚本目录执行（建 Junction 需要管理员权限）：

# 仅预览，不移动文件
powershell -ExecutionPolicy Bypass -File ".\Organize-PC.ps1"

# 实际执行
powershell -ExecutionPolicy Bypass -File ".\Organize-PC.ps1" -Apply
03

按菜单顺序操作

推荐顺序：扫描 → 预览方案 → 确认无误 → 执行迁移。

选项功能说明
1扫描磁盘 + 文件生成 inventory_*.json
2生成整理方案dry-run，生成 plan_*.json
3执行游戏迁移存档跟随
4资料归集初始化目标目录 + 迁移
5升级为真实分区高危 三次确认
6回滚上一次操作按日志反向恢复
config_builder

在线生成 config.psd1

脚本自动选剩余空间最大的盘
存放 models / datasets / docs / misc
小于此体积的目录不视为游戏（白名单除外）
