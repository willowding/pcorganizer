Administrator: Windows PowerShell
features

一个工具，搞定所有整理

GAME_DETECT

智能游戏识别

自动扫描 Steam、Epic、GOG、暴雪、育碧等主流启动器路径，结合体积 + exe + 目录结构启发式识别，无漏报。

AI_CLASSIFY

AI 资料分类归集

按文件后缀（.safetensors .gguf .pt …）和目录名关键字，自动区分模型权重、训练数据集、知识库文档四个类别。

SAVE_SYNC

存档跟随迁移

在 Documents、AppData、Saved Games 里模糊匹配存档目录，与游戏本体一起搬走，存档一条不丢。

JUNCTION

目录联接无感知

文件搬走后在原位建 NTFS Junction，启动器、游戏内部的路径全部不变，零配置即用。

DRY_RUN

先预览再执行

默认 dry-run 模式生成 plan.json 整理方案，审核无误后加 -Apply 参数再实际移动。

ROLLBACK

一键回滚

每次操作都写 op-log.json，随时反向执行：删 Junction → robocopy 复制回原位 → 清理新位置。

how_to_use

三步开始整理

01

下载 & 编辑配置

把 PCOrganizer 文件夹复制到目标电脑，用文本编辑器打开 config.psd1，确认游戏迁移盘符和 AI 资料根目录。也可以用下方的配置生成器自动生成。

# config.psd1 关键项
GameTargets    = @('D:\Games', 'E:\Games')
AIRoot         = 'E:\AI'
AIDriveLetter  = 'F'
GameMinSizeGB  = 1
02

以管理员身份运行

右键 PowerShell → 以管理员身份运行，进入脚本目录执行（建 Junction 需要管理员权限）：

# 仅预览，不移动文件（推荐先跑这个）
powershell -ExecutionPolicy Bypass -File ".\Organize-PC.ps1"

# 实际执行
powershell -ExecutionPolicy Bypass -File ".\Organize-PC.ps1" -Apply
03

按菜单顺序操作

推荐操作顺序：扫描 → 预览方案 → 确认无误 → 执行迁移。

选项功能说明
1扫描磁盘 + 文件生成 inventory_*.json
2生成整理方案dry-run，生成 plan_*.json
3执行游戏迁移存档跟随
4AI 资料归集初始化 AI 盘 + 迁移
5升级 AI 盘为真分区高危 三次确认
6回滚上一次操作按日志反向恢复
config_builder

在线生成 config.psd1

脚本自动选剩余空间最大的盘
存放 models / datasets / docs / misc
该盘符会映射到 AI 根目录，确保目标机器上该盘符未被占用
小于此体积的目录不视为游戏（白名单除外）
这些目录名的游戏将被跳过

ai_drive

AI 盘两阶段方案

没有空闲分区？没关系。先用虚拟盘顶着，等游戏搬走腾出空间再升级成真分区。

// 阶段一 · 立即可用

subst 虚拟盘

  • 在 E:\AI 下建 models / datasets / docs / misc
  • 用 subst F: E:\AI 映射虚拟盘
  • 写开机启动项，重启后自动恢复 F:
  • Ollama / ComfyUI / LM Studio 直接填 F:\models\xxx
// 阶段二 · 真实分区

压缩 E 盘新建分区

  • 检查 E 盘可压缩空间是否足够
  • 三次确认 → 压缩 E 盘 → 新建 NTFS 分区
  • 自动把 E:\AI 内容迁移到新分区根目录
  • 删除旧目录和 subst 开机项

// 阶段二操作前请先备份重要数据到外接存储
